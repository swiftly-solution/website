SwiftlyS2 provides a comprehensive translation system that allows you to localize your plugins for multiple languages. The system automatically loads translations from your plugin's resources folder and provides them to players based on their game language.

## Setting Up Translation Files

Translation files should be placed in your plugin's `resources/translations/` folder. Each file should be named using the language code followed by the `.jsonc` extension.

## Creating Translation Files

Translation files use the JSONC format (JSON with Comments), which allows you to add comments for context.

### Example: resources/translations/en.jsonc

```jsonc
{
  // Plugin information
  "plugin.name": "My Awesome Plugin",
  "plugin.version": "Version {0}",

  // Welcome messages
  "welcome.title": "Welcome!",
  "welcome.message": "Welcome to the server, {0}!",
  "welcome.first_time": "This is your first time here, {0}. Have fun!",

  // Commands
  "command.heal.success": "You have been healed!",
  "command.heal.other": "{0} healed {1}",
  "command.heal.no_permission": "You don't have permission to use this command.",

  // Notifications
  "notification.player_joined": "{0} has joined the server",
  "notification.player_left": "{0} has left the server",

  // Errors
  "error.not_found": "Player not found!",
  "error.invalid_argument": "Invalid argument: {0}"
}
```

### Example: resources/translations/fr.jsonc

```jsonc
{
  // Informations du plugin
  "plugin.name": "Mon Super Plugin",
  "plugin.version": "Version {0}",

  // Messages de bienvenue
  "welcome.title": "Bienvenue !",
  "welcome.message": "Bienvenue sur le serveur, {0} !",
  "welcome.first_time": "C'est votre première visite, {0}. Amusez-vous bien !",

  // Commandes
  "command.heal.success": "Vous avez été soigné !",

}
```

## Using Translations in Code

### Getting a Player's Localizer

The most common way to use translations is to get a localizer for a specific player, which automatically uses their game language.

```csharp
public void GreetPlayer(IPlayer player)
{
    var localizer = Core.Translation.GetPlayerLocalizer(player);
    var message = localizer["welcome.message", player.Controller.PlayerName];
    
    player.SendMessage(MessageType.Chat, message);
}
```

### Simple Translation Lookup

For simple translations without parameters:

```csharp
var localizer = Core.Translation.GetPlayerLocalizer(player);
var pluginName = localizer["plugin.name"];

player.SendMessage(MessageType.Chat, pluginName);
```

### Translation with Parameters

Use string formatting with parameters:

```csharp
var localizer = Core.Translation.GetPlayerLocalizer(player);
var message = localizer["command.heal.other", healer.Controller.PlayerName, target.Controller.PlayerName];

player.SendMessage(MessageType.Chat, message);
// Output: "PlayerA healed PlayerB" (or localized equivalent)
```

## Best Practices

### Keep Translation Keys Consistent

Use a consistent naming convention for your translation keys:

```
category.subcategory.specific_key
```

Examples:
- `command.heal.success`
- `command.heal.no_permission`
- `notification.player_joined`
- `error.invalid_argument`

### Provide Context with Comments

Use JSONC comments to provide context for translators:

```jsonc
{
  // {0} = player name
  "welcome.message": "Welcome, {0}!",

  // {0} = healer name, {1} = target name
  "command.heal.other": "{0} healed {1}",

  // Error shown when player is not found
  // {0} = searched player name
  "error.player_not_found": "Player '{0}' not found!"
}
```

### Always Provide English Fallback

Always create an `en.jsonc` file as the default fallback language. This ensures that if a player's language is not supported, they will see English text instead of missing translations.

### Use Parameters for Dynamic Content

Use numbered parameters (`{0}`, `{1}`, etc.) for dynamic content instead of string concatenation:

```jsonc
// Good
"welcome.message": "Welcome, {0}! You are player #{1}",

// Avoid
"welcome.message": "Welcome, " // Hard to translate properly
```

### Test with Multiple Languages

Test your plugin with different languages to ensure:
- All translation keys exist in all language files
- Parameters are used correctly
- Text formatting looks good in different languages (some languages are much longer)

### Keep Translations Synchronized

When adding new translation keys, add them to ALL language files. You can temporarily use English text as a placeholder until proper translations are available:

```jsonc
// fr.jsonc - Temporary English placeholder
"new.feature.message": "This is a new feature", // TODO: Translate to French
```

## Handling Missing Translations

If a translation key is missing, the system will return the key itself. You can check for this and provide a fallback:

```csharp
var localizer = Core.Translation.GetPlayerLocalizer(player);
var message = localizer["some.key"];

// If translation is missing, message will be "some.key"
if (message == "some.key")
{
    message = "Default fallback message";
}

player.PrintToChat(message);
```
